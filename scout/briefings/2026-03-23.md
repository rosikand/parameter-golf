### Technique Combos That Help

1. **SwiGLU + U-Net gated skips + AdamW TTT**: The standout finding from PR #462. TTT gain is 0.053 BPB vs 0.019 on standard arch — the loss landscape is fundamentally smoother. Mechanism: gated residual paths reduce gradient sharpness in the quantized model.

2. **EMA + Tight SWA**: These are complementary, not redundant. SWA provides checkpoint diversity; EMA at every step provides smooth exponential tracking. Combined −0.0012 BPB vs either alone.

3. **GPTQ + Extended Warmdown (3500 steps) + Late QAT (0.15 threshold)**: The #414 stack. Each piece contributes additively: GPTQ (−0.0006), EMA (−0.0006), warmdown (−0.0002), QAT (−0.0001).

4. **Cosine TTT + Per-layer LR groups**: PR #481 shows that MLP output projections have 3.4× higher relative quantization error and need 3× higher TTT LR. This targeted allocation avoids undershooting noisy layers.

5. **Value Residual + Gated Attention + AdamW TTT**: PR #490's combination — small parameter additions (+2 params/layer for VR, +4104/layer for gates) improve TTT effectiveness.

### Technique Combos That Hurt

- **XSA removal for step-time savings**: PR #398 removed XSA to save ~1.4ms/step, but subsequent entries show XSA is worth significantly more than the training time it costs.
- **Late QAT + Aggressive TTT**: PR #398 found these antagonistic — late QAT compresses the quantization gap that TTT then tries to undo.
- **Sequence length curriculum + SWA**: PR #140 documents that seq curriculum breaks SWA (SWA checkpoints trained at different seq lengths can't be meaningfully averaged).
- **LoRA TTT (standard approach)**: LoRA TTT in the top entries (PR #442 baseline) yields only −0.004 BPB; AdamW TTT on all params beats it substantially. The LoRA rank-8 approach from issue #83 is now dominated.
- **PR #505's seq_len=2048 training**: Gains −0.009 BPB but costs the entire training budget headroom (18 min/seed). Not viable within 10-min constraint without architectural compression.

### Emerging Techniques Worth Watching

1. **TrigramHash (PR #486)**: 4096 buckets, dim=128, xor of three token hashes. Preliminary results show it adds context but at 22% training step overhead — needs a faster implementation.
2. **GradQuant / Adaptive quantization (PR #486)**: Sensitivity-guided per-tensor bitwidth assignment (Int7/6/5 by gradient sensitivity). Currently costs training throughput; if the sensitivity map can be precomputed cheaply, this could be free.
3. **Parallel Muon / Parameter Banking (PR #473)**: 82.3ms/step is competitive; batched Newton-Schulz reduces optimizer overhead and could enable another ~100 extra training steps in 600s.
4. **All-layer XSA (PR #503)**: XSA on all 11 layers instead of last 4 — worth testing carefully since #503 reports a good result (1.1218 mean).
5. **Depth-scaled residuals (PR #512)**: 1/√(layer+1) scaling for stability — plausible mechanism, but needs a trustworthy implementation to evaluate.
