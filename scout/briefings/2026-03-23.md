### Technique Combos That Help

- **EMA + AdamW TTT**: The single most impactful pairing in the legitimate leaderboard. EMA provides a better starting point for TTT; AdamW TTT (lr=0.0001–0.0005, 3–10 epochs) outperforms SGD TTT by ~0.019 BPB while taking 40% less time (#442 vs prior SGD runs).
- **GPTQ-lite + Int6 QAT**: These are complementary, not redundant. QAT reduces fake-quantization error during training; GPTQ-lite minimizes real quantization error at export. Together they compress the quant gap to ~0.001–0.016 BPB.
- **Tight SWA + EMA**: PR #414 shows both can coexist (+0.0006 BPB each). Key constraint: EMA must use decay=0.997 to avoid the 0.14 BPB quant gap observed on SWA stacks (issue #140).
- **Value Residual + AdamW TTT**: PR #490's 1.0891 result — only one seed but the combination has clear mechanistic justification (V residual stabilizes gradient flow for TTT fine-tuning).

### Technique Combos That Hurt or Are Neutral

- **Sequential TTT + SWA-only**: SWA checkpoints do not benefit from additional training steps (confirmed by #399 analysis). Only EMA-based submissions gain from faster per-step times.
- **TrigramHash + GradQuant overhead**: PR #486 achieves strong BPB (1.1132) but uses 22% fewer training steps due to compute overhead from adaptive quantization. The net gain is positive but diminishing.
- **MTP (multi-token prediction)**: Tested in issue #375 — added +0.028 BPB penalty. Not recommended.
- **Full-run QAT (vs Late QAT)**: Issue #375 confirms full-run QAT hurts vs late QAT. Threshold 0.15 (PR #414) is the current sweet spot.
- **XSA on all 11 layers (PR #503) vs XSA on last 4 (PR #417)**: Both achieve ~1.122 range; full-layer XSA is not clearly better than XSA4, and introduces a documented RoPE sequence length mismatch bug.

### Emerging Techniques Worth Watching

1. **Value Residual (ResFormer)**: Present in both #490 (best single-seed) and #486 (strong 3-seed). Low parameter cost (+2 scalars/layer for 11L = 22 params total). Mechanistically sound — caches first-block V to stabilize deep layer gradients. **High priority to validate.**
2. **Gated Attention**: PR #490 adds per-head sigmoid gate (init bias=4.0, +4104 params/layer). Combined with Value Residual, contributes to the 1.0891 result. Untested in isolation — unclear which component drives the improvement.
3. **TrigramHash (4096 buckets, dim=128)**: Upgrade from BigramHash(2048) in PR #486. Provides richer n-gram context at modest cost. −0.023 BPB vs baseline when combined with other techniques.
4. **AdamW TTT with cosine scheduling**: PR #486 uses cosine LR in TTT rather than flat LR. Marginal but consistent improvement signal.
5. **Parallel Muon / Parameter Banking**: PR #473 achieves 82.3ms/step. Primarily a throughput play; ~0.001 BPB improvement from extra steps. Low complexity, easy to adopt.

## Frontier Gaps & Opportunities

Ranked by expected impact (highest first):

### 🔴 Tier 1: High Expected Impact (try immediately)

**1. Value Residual + Gated Attention + AdamW TTT + multi-seed confirmation**
- PR #490 achieves 1.0891 on a single seed. Run 2 more seeds to confirm. If the mean holds near 1.089–1.092, this beats #442 (1.1027) by ~0.013 BPB.
- Cost: 2 × 600s = 20 GPU-minutes on 8xH100. Extremely high ROI.
- Specific: use PR #490's exact config (EMA=0.997, AdamW TTT lr=5e-4, 10ep, last 2 blocks + norms + lm_head unfrozen, sliding window stride=64).

**2. Value Residual + Gated Attention + TrigramHash (replacing BigramHash)**
- PR #486 shows TrigramHash (4096 buckets) gives a clean improvement over BigramHash(2048). PR #490 still uses BigramHash.
- Swapping BigramHash → TrigramHash in the #490 architecture is a 1-line change with ~−0.005 expected BPB based on #486's delta.
- **Untested combination**: Value Residual + Gated Attention + TrigramHash + AdamW TTT. Expected range: **1.082–1.086 BPB**.

**3. GradQuant (adaptive Int5/6/7) applied to Value Residual + Gated Attention baseline**
- PR #486's GradQuant saves bits on low-sensitivity parameters, enabling a slightly larger model within 16MB.
- Currently no submission combines GradQuant with Value Residual. The expected saving is ~0.5–1MB of artifact space, potentially allowing 1–2 extra layers or wider MLP.
- Expected impact: **−0.004 to −0.008 BPB** if space reclaimed goes into model capacity.

### 🟡 Tier 2: Medium Expected Impact (high confidence, moderate effort)

**4. GPTQ-lite applied to PR #490 architecture**
- GPTQ-lite gives −0.0006 BPB at zero training cost (confirmed in #414, #379). PR #490 does not mention it.
- Zero-risk add-on. Expected: **−0.0006 BPB**, bringing 1.0891 → ~1.0885.

**5. Two-phase TTT (norm recalibration + block adaptation) on EMA+Value Residual base**
- PR #417's two-phase TTT separates quantization-artifact correction (phase 1: norm-only, 50ep, lr=0.01) from distribution adaptation (phase 2: last 3 blocks, 10ep, SGD lr=0.005).
- Currently no submission combines two-phase TTT with Value Residual + Gated Attention.
- Phase 1 specifically targets the quant artifact problem — highly complementary with GradQuant's adaptive precision.
- Expected: **−0.005 to −0.010 BPB** over single-phase AdamW TTT.

**6. FA3 Hopper + Parallel Muon on PR #490 architecture**
- PR #490 runs at 99.66ms/step. PR #473 achieves 82.3ms/step with Parallel Muon.
- Delta: ~17ms/step → ~120 extra training steps in 600s → approximately **−0.006 BPB** (using issue #375's 0.006 BPB/ms estimate).
- Note: EMA-based submissions benefit from extra steps; this applies here.

**7. Warmdown extension to 3500 steps on #490-class architecture**
- PR #414 shows warmdown 3500 (vs default) gives −0.0002 BPB. Cheap, confirmed, not yet in #490.

### 🟢 Tier 3: Speculative / Underexplored

**8. AdamW TTT with per-layer learning rate scaling**
- Current TTT implementations use a flat LR across all unfrozen layers. Deeper layers typically benefit from smaller LRs during fine-tuning. A 0.5× LR multiplier on earlier unfrozen layers could reduce catastrophic interference.
- No submission has tried this. Expected: **−0.001 to −0.003 BPB** at negligible implementation cost.

**9. TrigramHash + Value Residual without TTT (non-TTT SOTA attempt)**
- The best non-TTT result is PR #414 at 1.1233. Adding TrigramHash and Value Residual (both validated positives in #486) to the #414 stack could push non-TTT SOTA to ~**1.115–1.118**.
- Useful as a clean ablation and competitive if TTT overhead is a concern.

**10. muP (maximal update parameterization) on the current SOTA stack**
- muP appears in PR #262 (ruled invalid for other reasons) but has avg_bpb of 1.1032 — well above the frontier. Only 3 submissions use it.
- Worth re-testing on the current 11L EMA+TTT stack. muP's LR scaling could enable larger batch sizes or higher peak LRs without instability. **High uncertainty, moderate upside.**

**11. QK-Norm on the current stack**
- Only appears in PR #216 (ternary, suspicious). QK-Norm (normalizing Q and K before attention) is well-supported in the literature for training stability and has essentially zero parameter cost. It is conspicuously absent from all top legitimate submissions.
- Expected: **−0.001 to −0.004 BPB**, particularly beneficial in deep (11L) models where attention logit variance can cause instability.
