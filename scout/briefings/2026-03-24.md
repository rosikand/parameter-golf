### Technique Combos That Appear to Help
- **LoRA TTT + depth-scaled residual + cosine LR**: The PROTEUS/DeepQuant lineage (#512→#568→#596) shows each incremental addition yields significant gains. This stack is the current best direction.
- **SwiGLU + U-Net + AdamW TTT**: JoeProAI (#462) found this combo produces 2.8× more TTT lift than a standard architecture (Δ0.053 vs Δ0.019), suggesting smoother loss geometry.
- **Full Hessian GPTQ + LeakyReLU²**: PR #593 shows GPTQ as a net positive without any TTT; combining with TTT is unexplored.

### Technique Combos That Appear to Hurt or Have Issues
- **Multi-epoch global TTT** (non-causal): Formally invalid per #402. PRs #481, #517, #532 all closed. Any submission with per-document reset or score-first ordering is required.
- **LoRA TTT > 600s total eval**: PR #596 demonstrates the hard ceiling — without time limit it achieves 0.5684 vs 0.6430 with fallback. Long-document fallback is a significant hidden cost.
- **High-rank LoRA without time budget analysis**: Rank-16 in #596 pushes close to the 600s limit. Rank-32 or more TTT epochs will likely exceed it without architectural optimization.

### Emerging Techniques Worth Watching
1. **Per-block bias tuning during TTT** (PR #596): Free parameters that shift layer outputs without touching quantized weights. Likely generalizes beyond LoRA.
2. **Post-TTT temperature calibration** (PR #576, #596): T=0.98 corrects systematic overconfidence from TTT; -0.003 BPB at zero compute. Should be universally adopted.
3. **Codebook + Huffman compression** (PR #532, closed for TTT violation): The compression pipeline itself (21% smaller than Int6+Zstd) is valid and could enable larger models. Re-opening on a compliant TTT base is high-value.
4. **SharedSparseSidecar** (PR #555, #581): Late-layer sparse auxiliary module with per-site conditioning shows ~-0.022 BPB in ablation. Not yet combined with LoRA TTT.
5. **TrigramHash** (PR #486): Small improvement over BigramHash, unexplored in top LoRA TTT submissions.

## Frontier Gaps & Opportunities

Ranked by estimated impact:

### 🔴 Tier 1 — High Expected Impact

**1. LoRA TTT time efficiency: faster iterations within 600s**
- PR #596 identifies that long documents cause fallback to base model, costing ~0.075 BPB on the tail. The gap between unconstrained (0.5684) and constrained (0.6430) is the largest single opportunity in the competition.
- **Experiment**: Profile which document lengths exceed per-doc TTT budget. Implement adaptive epoch count per document (more epochs for short docs, fewer for long). Expected gain: **-0.02 to -0.05 BPB**.
- **Experiment**: Quantize LoRA adapters themselves (Int8 gradients during TTT) to speed up per-step time, enabling more epochs or higher rank within budget.

**2. LoRA TTT rank scaling with smarter budget allocation**
- #596 uses rank-16 globally. Rank may be better concentrated in specific layers (e.g., higher rank for layers with highest quantization error, as done by per-layer LR in #481).
- **Experiment**: Rank-32 on top 3 layers + rank-8 elsewhere, keeping total param count same as rank-16 uniform. Expected gain: **-0.01 to -0.02 BPB**.

**3. Depth-scaled residual (1/√(layer+1)) on non-PROTEUS architectures**
- Only 2 PRs use depth-scaled residuals despite them appearing in top-2 legitimate submissions. No submission has tried this on the SwiGLU+U-Net+XSA stack (#462 architecture).
- **Experiment**: Add depth-scaled residual scaling to JoeProAI's #462 architecture + LoRA TTT. Expected gain: **-0.01 to -0.03 BPB** (interpolating from PROTEUS results).

**4. Codebook+Huffman compression on a compliant TTT base**
- PR #532 achieved 14.12MB artifact vs 18+MB with Int6+Zstd, enabling a larger model. It was closed only for TTT rule violation, not for the compression technique.
- **Experiment**: Apply CB-48/CB-80/CB-64 codebook quantization + Huffman + Zstd to the PROTEUS/DeepQuant base model (11L Int6 LoRA TTT stack). The 21% size reduction could allow 11L→13L at the same file size, or larger hidden dim. Expected gain: **-0.01 to -0.03 BPB**.

- Expected gain from architecture swap alone: **-0.01 to -0.02 BPB** on top of current LoRA TTT baseline.
